# Real import execution

The governed real import uses streaming `Microsoft.Data.SqlClient.SqlBulkCopy` for FDA and Medicaid raw rows. Supported bounded batch sizes are 20,000, 50,000 (default), and 100,000. Each batch uses `EnableStreaming`, explicit column mappings, `TableLock`, constraint checking, an unlimited long-running timeout, and one explicit transaction containing both the bulk rows and checkpoint/audit update. Raw entities never enter the EF change tracker; EF remains responsible for DatasetVersion, SourceFile, registrations, import runs, exclusions, mapping review, checkpoints, audits, and reconciliation.

The new-import entry point is `scripts/Invoke-ResearchDataImport.ps1`. Resume uses `scripts/Resume-ResearchDataImport.ps1`, the original import correlation to locate the run, and a distinct resume-audit ID. It verifies the exact database and ownership, approved protocol, non-final dataset, manifest/source hashes, schema profiles, checkpoint/source relationships, and absence of raw rows beyond checkpoints. Confirmed resume requires a clean worktree. A dirty-worktree exception exists only for `-WhatIf` review.

Both raw tables retain their clustered identity primary keys, unique `(SourceFileId, SourceRowNumber)` lineage indexes, and `(SourceFileId, ParseStatus)` indexes. The unique lineage indexes are required for duplicate defense during resume. The status indexes remain useful for post-load quality review. Disabling and rebuilding them during a partially completed governed run would add recovery risk, so this optimization does not alter indexes.

The import never automatically finalizes a DatasetVersion, builds features, trains a model, estimates a causal effect, or approves a research freeze. Completion first reconciles committed database rows against committed source-row checkpoints.
